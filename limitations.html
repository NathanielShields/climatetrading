Limitations & honest disclaimers

An MVP backtest is useful for designing the system and validating the plumbing — but it is not yet a tradeable strategy. This page lists every place where the current implementation is approximate, mocked, or otherwise different from production.

What's actually mocked

MOCK Demo data path

The headline demo run uses the mock data builder and the mock forecast provider. It exists to demonstrate the full pipeline with zero external dependencies — not to claim historical realism. Treat its $114k gross P&L as a plumbing-test artifact, not a strategy result.

READY (GPU) GraphCast / Earth-2

Both providers now ship a four-tier fallback (docker → in-proc → file → mock) and a matching docker/earth2/Dockerfile built on nvcr.io/nvidia/modulus. Real FourCastNet and GraphCast-small runs happen in the container; the Python host only speaks CSVs. Activated with AG_VOL_ARB_DOCKER_FORECASTS=1 + NGC_API_KEY + an RTX-class GPU reachable from Docker Desktop's WSL2 backend. Full run-through on the models page →

What's approximated

PROXY ATM IV

True ATM IV requires the live CME corn options chain, which sits behind a CME DataMine subscription (paid). We approximate it via Yang-Zhang realised vol + drought-regime calibration premium + term-structure premium. Empirically corn ATM IV/HV runs 1.20–1.40× in normal markets and 1.5–2.0× in stress regimes; the calibration sits inside this range. Not a substitute for real chain data when sizing capital.

REAL Daily bars only

Everything is collapsed to daily resolution. Real options markets show intraday vol crush around USDA reports, weather forecasts, and Sunday-night gaps. The MVP can't model any of that — every signal is evaluated against daily settle.

REAL Single-state proxy

Iowa is treated as a stand-in for the entire U.S. corn belt. In reality the production-weighted average across IA + IL + IN + NE + MN matters, and a regional decoupling (e.g. Iowa drought + Illinois floods) would dilute a pure-Iowa signal.

REAL One event per year

The current engine returns the first qualifying drought event in the simulation window. Multi-event detection inside one growing season (rare but possible — e.g. early-July dry spell, then late-August drought) is a natural extension, not yet implemented.

PROXY Slippage model

Tick-based bid-ask + commissions + market-impact widening. Calibrated to typical CME corn option spreads but not to actual fills. Real execution would also depend on time of day (open/close vs midday) and on whether a USDA report is imminent.

REAL Pricing model

Black-76 with constant rates, no dividends, no stochastic vol. Adequate for ATM short-dated straddles but doesn't capture skew or the term structure of vol — both of which a serious vol trader would care about, especially for partial exits.

What we're not modelling at all

Funding and margin financing

Long options have no CME margin requirement beyond premium, but the premium itself is capital deployed for weeks. The current model uses a single risk-free rate; it doesn't model margin financing costs, opportunity cost of capital tied up, or the capital efficiency of structured alternatives (call ratios, calendar spreads).

Tax

Long options held across calendar boundaries can have non-trivial Section 1256 treatment for U.S. taxpayers (60/40 long/short term split). All P&L numbers are pre-tax.

Counterparty & clearing

CME clearing eliminates direct counterparty risk but introduces FCM / clearing-broker risk and concentration limits. Not modelled.

News / report calendar awareness

WASDE, Crop Production, Quarterly Stocks reports cause large jumps in corn futures and IV. The signal engine ignores the reporting calendar; production strategy would almost certainly want to side-step holding into known event-risk windows.

Position sizing dynamics

The MVP enters a fixed 100-contract position. A real strategy would size based on signal conviction, vol regime, available capital, and a Kelly-derived max-utilisation cap. The MarginAccount can enforce the cap but doesn't drive sizing.

What's actually production-ready

Don't read the above as "everything is mocked." The pieces below are real, tested, and exercised in the multi-year sweep:
  • Anti-lookahead infrastructure — every adapter stamps both timestamps; the engine filters strictly.
  • Open-Meteo ERA5 weather + soil — real data, no key required, used in multi-year sweep.
  • yfinance ZC=F continuous futures — real OHLC, used in multi-year sweep.
  • Yang-Zhang + Garman-Klass + IV calibration — real estimators (not proxies for plumbing), with full unit-test coverage.
  • Black-76 + Greeks — standard implementation, regression-tested.
  • CME contract calendar — H/K/N/U/Z, last-Friday expiry, 30-day roll trigger.
  • Tick-based bid-ask model — IV-dependent, time-to-expiry-dependent, capped, scales with order size.
  • Margin account — capital deployment tracking with soft cap.
  • Full-re-pricing P&L attribution — exact decomposition (futures + vol + theta + cross-gamma sums to total to floating-point precision).
  • NASA Earthdata adapters — credential-guarded but functional; HDF4 + HDF5 NDVI both verified.
  • USDA NASS adapter — key-guarded; falls back to mock with a clear warning.
