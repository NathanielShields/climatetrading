Reference

Reproducibility

Every artifact on this site, plus the figures, plus the multi-year sweep, can be regenerated from a clean clone in under five minutes (assuming Python ≥ 3.11).

Test status

Quickstart

# clone & install
git clone <repo-url> ag_vol_arb
cd ag_vol_arb
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# verify everything works
PYTHONPATH=src python -m pytest -q
PYTHONPATH=src python -m ag_vol_arb.cli smoke-test

# run the demo (mock data, ~5 seconds)
PYTHONPATH=src python -m ag_vol_arb.cli run-demo

# run the multi-year sweep (real data, ~2 minutes)
PYTHONPATH=src python -m ag_vol_arb.cli run-historical \
  --years 2012,2019,2022,2023 \
  --config historical.yaml

What you get

outputs/demo/

  • trade_blotter.csv — entry / partial / exit rows
  • daily_marks.csv — daily MTM through the trade
  • attribution.csv — daily P&L decomposition
  • event_summary.csv — one-row event summary
  • aggregate_summary.csv — headline metrics
  • pnl_timeseries.png, attribution.png
  • report.md — human-readable summary

outputs/historical/

  • multi_year_aggregate.csv — one row per year
  • 2012/, 2019/, 2022/, 2023/ — per-year artifacts (same shape as demo)

Refreshing the website after a new run

# 1. run a new backtest
PYTHONPATH=src python -m ag_vol_arb.cli run-demo
# (or run-historical, or anything that writes to outputs/)

# 2. export the artifacts into the site's data folder
PYTHONPATH=src python scripts/export_site_data.py

# 3. preview locally
python -m http.server -d docs 8080
# open http://localhost:8080/

# 4. commit & push — GitHub Pages serves /docs automatically
git add docs/
git commit -m "site: refresh after new backtest run"
git push

Deploying the website to GitHub Pages

The docs/ folder is the GitHub Pages source. No build step is required — just enable Pages in repo settings.

Option A — settings-based (no Action required)

  1. Go to Settings → Pages in your repo.
  2. Under Build and deployment, set Source to Deploy from a branch.
  3. Choose branch main and folder /docs.
  4. Click Save. First deploy takes ~30 seconds.
  5. Site lives at https://<your-username>.github.io/<repo-name>/.

Option B — GitHub Action (recommended)

The repo ships a workflow at .github/workflows/pages.yml that re-exports site data and publishes docs/ on every push to main. Enable it under Settings → Pages → Source = GitHub Actions.

What the workflow does 
1. Checkout repo
2. Set up Python
3. Install requirements + run pytest
4. Run scripts/export_site_data.py to refresh docs/assets/data/
5. Upload docs/ as a Pages artifact
6. Deploy

GPU forecast pipeline (optional)

Adds real GraphCast / FourCastNet inference as a drop-in for the file/mock providers. Requires WSL2 + Docker Desktop (NVIDIA Container Toolkit) + a CUDA-capable GPU (≥4 GB VRAM). 
# 0. inside WSL2 — verify nvidia-smi works inside docker
bash scripts/check_gpu_stack.sh

# 1. build the inference image (one-time, ~15 minutes; ~12 GB)
docker build -t ag-vol-arb/earth2:latest -f docker/earth2/Dockerfile .

# 2. smoke-test one forecast end-to-end
bash scripts/check_gpu_stack.sh --smoke

# 3. pre-bake a season of FourCastNet forecasts (one container call per issue date)
python scripts/run_weather_docker.py batch \
    --model fcn --start 2025-05-01 --end 2025-09-30 --step 7 \
    --region iowa --lat 41.6 --lon -93.6 \
    --out outputs/forecasts/fcn_iowa.csv

# 4. run a backtest that consumes them — provider auto-stamps fcn:real
export AG_VOL_ARB_DOCKER_FORECASTS=1
export NGC_API_KEY=nvapi-...
PYTHONPATH=src python -m ag_vol_arb.cli run-backtest --config configs/real_gpu.yaml

Switch --model fcn to --model graphcast for GraphCast-small. Same image, same orchestrator.
