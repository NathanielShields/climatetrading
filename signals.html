Why "two-of-N" instead of a fitted model?

The MVP uses a simple confirmation rule: at least one initial signal + one confirming signal must fire on the same day. This is intentionally conservative — it trades a little signal-to-noise for a lot of robustness. With only 4 strong drought events in the available training window (2012, 2019 marginal, 2022, 2023), there's not enough sample size to fit a logistic model without overfitting.

What the production version would do

  • Replace the boolean rule with a logistic model fit on the full 1980–2024 USDA + ERA5 history.
  • Use the GraphCast/Earth-2 ensemble distribution to get a true probability of drought progression, rather than a single deterministic forecast value.
  • Calibrate per state (Iowa vs Illinois vs Nebraska have different USDA condition baselines).
  • Add a soft Kelly sizing layer instead of fixed 100-contract entries.
