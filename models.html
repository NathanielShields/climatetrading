Forecast model layer

Every forecast provider conforms to one interface, so swapping in a real GraphCast or Earth-2 inference run is a drop-in replacement for the mock during research.

The ForecastProvider interface

class ForecastProvider(Protocol):
    name: str
    def fetch(self, as_of: date, region: str, horizon_days: int) -> ForecastRecord:
        ...

@dataclass
class ForecastRecord:
    issue_ts: date              # when the forecast was made
    available_ts: date          # when a trader could see it
    horizon_days: int           # 10, 15, or 16 typically
    region: str                 # 'iowa'
    prob_rain_over_1in_10d: float
    precip_forecast_in: float
    temp_max_forecast_f: float
    provider: str               # 'open_meteo' | 'graphcast' | 'earth2' | 'mock'

Why GraphCast / Earth-2 matter for this strategy

The vol mispricing window

Corn options are written by commercials whose hedging desk sees soil and weather just like everyone else — but the option premium reflects the distribution of expected rainfall, not the modal forecast. AI weather models (GraphCast, Earth-2/FourCastNet, Pangu-Weather) materially outperform operational NWP at 5–10 day horizons on ECMWF benchmarks, which is exactly the window where pollination-stress decisions get made.

If you can see a sharper distribution earlier, you can buy vol before the options desk re-prices.

Where they slot in

The system's signal engine consumes prob_rain_over_1in_10d as a confirmation gate. With Open-Meteo (ECMWF-derived) you get a single forecast value; with GraphCast or Earth-2 ensembles, you get a forecast distribution and can use the full ensemble for a sharper probability, lower variance, and better-calibrated uncertainty cones.

Four-tier fallback (graceful degradation)

Each provider tries the real Docker path first, then in-process inference, then a file-backed snapshot, then the deterministic mock. Same code path regardless of which tier serves the request — the engine never branches on which mode produced the forecast. The only difference is the provider stamp on the record (fcn:real / graphcast:real / earth2:file / earth2:fallback …).

Tier 1 — Docker GPU inference

Spawns the ag-vol-arb/earth2 container on the host, mounts a cache dir, runs FCN or GraphCast-small from live GFS analysis, writes a one-row CSV.

Status on this build: READY (GPU)

Activate with AG_VOL_ARB_DOCKER_FORECASTS=1 + docker build -t ag-vol-arb/earth2:latest -f docker/earth2/Dockerfile . in WSL2.

Tier 2 — In-process earth2studio

Fast path for users with earth2studio + eccodes already in the same Python env (Linux/conda). Skips the docker overhead.

Status: OPT-IN

Triggers when NGC_API_KEY set and earth2studio imports cleanly in-proc.

Tier 3 — File-backed

Reads a pre-baked CSV produced by scripts/run_weather_docker.py batch. Lets you run inference once on a beefy box and replay against many backtests.

Status: READY

Drop a CSV at outputs/forecasts/<provider>_<region>.csv.

Tier 4 — Mock

Deterministic synthetic forecast keyed on date + region. Always-on safety net so the entire pipeline works with zero external dependencies. Every CI test uses this.

Status: ALWAYS ON

End-to-end on your GPU box

# 1. Verify the stack (run inside WSL2)
bash scripts/check_gpu_stack.sh

# 2. Build the inference image (one-time, ~15 min)
docker build -t ag-vol-arb/earth2:latest -f docker/earth2/Dockerfile .

# 3. (Optional) Smoke-test one forecast
bash scripts/check_gpu_stack.sh --smoke

# 4. Pre-bake a season of weekly forecasts (FourCastNet)
python scripts/run_weather_docker.py batch \
    --model fcn --start 2025-05-01 --end 2025-09-30 --step 7 \
    --region iowa --lat 41.6 --lon -93.6 \
    --out outputs/forecasts/fcn_iowa.csv

# 5. Run a backtest that consumes them
export AG_VOL_ARB_DOCKER_FORECASTS=1
export NGC_API_KEY=nvapi-...
PYTHONPATH=src python -m ag_vol_arb.cli run-backtest --config configs/real_gpu.yaml

Hardware envelope (what fits where)

RTX 4060 (8 GB) — what works
  • FourCastNet base (26 channels, ~74 M params): ✓ ~30 s / forecast
  • GraphCast-small (ONNX-RT): ✓ ~45 s / forecast
  • Full DeepMind GraphCast (~36 M params, JAX): borderline — works at fp16 with care
  • SFNO / Pangu-Weather: ✓
Software stack baked into the image
  • Base: nvcr.io/nvidia/modulus/modulus:24.12 (CUDA 12.5, PyTorch 2.4)
  • earth2studio >=0.13 (FCN, GraphcastSmall, SFNO, Pangu loaders)
  • eccodes C library + pygrib (preinstalled in base)
  • jax[cuda12] + dm-haiku for future upstream-GraphCast swap
  • Python deps locked in docker/earth2/Dockerfile
Initial conditions (no CDS account needed)
  • Default: NOAA GFS analysis via earth2studio.data.GFS — free, no auth
  • Alternative: NCEP GDAS via --ic-source gdas
  • Alternative: ECMWF IFS via --ic-source ifs (key required)
  • Pre-staged Zarr: --ic-zarr /work/ics/foo.zarr
Caching
  • Per-issue CSVs cached at ~/.cache/ag_vol_arb/forecasts/<model>/<region>/<date>.csv — second call for the same date is free
  • NGC weight cache mounted at ~/.cache/earth2 — survives container teardown
  • HuggingFace mirror: set HF_TOKEN for gated checkpoints
